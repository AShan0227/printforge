# PrintForge 单位经济模型

## 成本结构（每次生成）

| 项目 | 成本 | 来源 |
|------|------|------|
| Hunyuan3D-2 推理 | **$0** | HF Space 免费 |
| 带宽（上传图片+下载 GLB） | ~$0.001 | ~20MB 传输 |
| 服务器计算（watertight + 优化） | ~$0.002 | ~2s CPU |
| **总成本/次** | **~$0.003** | |

**注意：** HF Space 免费是当前状态。如果 Space 关闭或限流，需要自建 GPU 推理：
- AWS g5.xlarge: $1.006/h → 每次推理 ~42s → **$0.012/次**
- 月 10K 次: **$120/月**

## 关键指标

| 指标 | 目标 |
|------|------|
| Free → Pro 转化率 | 10% |
| 月活用户 (MAU) | 5,000 (6 个月内) |
| 用户获取成本 (CAC) | $0 (开源+社区) |
| 用户生命周期价值 (LTV) | $59.4 (Pro × 6 个月) |
| 盈亏平衡点 | **92 个 Pro 用户** |

## Dalio 原则
> "92 个付费用户就能盈亏平衡。这是一个非常低的门槛。关键是：第一批 92 个 Pro 用户从哪里来？" 
> 答案：Reddit r/BambuLab (120万拓竹用户) + MakerWorld + Product Hunt launch
