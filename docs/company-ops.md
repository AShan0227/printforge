# PrintForge 公司运营手册 v2

## 组织架构（自治型）

每个角色是独立的工作单元，拥有：
- 自己的**知识库**（领域专业知识）
- 自己的**skill**（可复用的工具/流程）
- 自己的**待办清单**
- **自主判断权**——不需要 CEO 批准每个细节

CEO 只做：战略方向 + 资源冲突仲裁 + 最终产品决策

---

## 各角色自治任务

### ⚡ Musk (CTO) — 产品工程
**知识库：** 3D 重建算法、mesh 处理、打印优化、系统架构
**当前 skill：** TripoSR/Hunyuan3D 推理、watertight pipeline、trimesh 操作
**独立开发清单：**
1. 实时打印预测 — AI 预测打印是否会失败（翘曲/层裂/支撑断裂）
2. 自适应 mesh 密度 — 平面区域低面数，曲面区域高面数
3. 多打印机配置文件 — 不同打印机参数预设（X1C/P1S/A1/A1 Mini/Prusa/Ender）
4. WebGL 3D 编辑器 — 浏览器内直接修改模型（移动/旋转/缩放/切割）
5. gcode 预览 — 可视化打印路径，无需 Bambu Studio

### 🔬 Feynman (研究员) — 技术前沿 + 用户洞察
**知识库：** 竞品技术栈、用户行为模式、社区动态
**当前 skill：** web_search、Reddit/HN 监控
**独立开发清单：**
1. 竞品监控系统 — 每周自动抓取 Meshy/Tripo/Hitem3D 的更新和定价变化
2. 用户反馈分析 — 从 GitHub Issues + Reddit 评论提取痛点
3. SEO 内容矩阵 — 为每个目标关键词写一篇针对性文章
4. 社区回复模板 — 当用户问"怎么把照片变成 3D 打印"时的标准回复

### 📊 Dalio (分析师) — 数据 + 商业策略
**知识库：** 市场数据、定价模型、单位经济、用户转化
**当前 skill：** 五维评估、原则提炼
**独立开发清单：**
1. 用户漏斗模型 — 从 GitHub Star → Clone → Install → First Use → Repeat → Pro
2. A/B 测试框架 — 在 Web UI 中嵌入简单的实验系统
3. 收入预测模型 — 基于 MAU 增长曲线预测 12 个月收入
4. 竞品定价跟踪看板 — Meshy/Tripo/Hitem3D 的价格变化历史

### ⚔️ Munger (风控) — 安全 + 法律 + 质量
**知识库：** 风险登记册、法律模板、质量基线
**当前 skill：** 逆向思维、Grill Me 协议
**独立开发清单：**
1. NSFW/版权检测模块 — 拦截不当内容（Munger R2 缓解）
2. 质量基线库 — 收集 50 张测试图片，建立质量评分基线
3. 回滚系统 — 如果新版本质量下降，自动回滚到上一版
4. 安全审计自动化 — 定期扫描依赖漏洞 + API 安全检查

### ✍️ Graham (写作) — 内容 + 传播 + 品牌
**知识库：** 发布文案库、教程模板、SEO 写作规范
**当前 skill：** Show HN/Reddit/PH 文案
**独立开发清单：**
1. 产品文档站 — 用 Mintlify/Docusaurus 搭建文档网站
2. 视频脚本 — 3 个 YouTube 短视频脚本（30s/1min/5min）
3. 邮件模板 — 新用户欢迎邮件 + 功能更新通知
4. 案例库 — 收集 10 个"我用 PrintForge 打印了 X"的真实案例

### 🔬 Feynman-X (跨界) — 创新 + 新技术集成
**知识库：** 3DGS、NeRF、扩散模型、跨学科连接
**当前 skill：** 跨领域类比、技术路线分析
**独立开发清单：**
1. 3DGS→Print 原型 — 实现 video → 3DGS → mesh → PrintForge 的 MVP
2. LiDAR 扫描集成 — iPhone 12 Pro+ 的 LiDAR 数据 → mesh
3. AR 预览功能 — 打印前在手机上 AR 预览实际大小
4. 物理仿真 — 简化 FEA 预测打印件能承受多少重量

## 协作机制

### 自发讨论（不需要 CEO 批准）
- Musk × Feynman-X: 新技术可行性评估
- Feynman × Graham: 内容策略对齐（研究发现 → 文章选题）
- Dalio × Munger: 风险对冲（新功能的 ROI vs 风险）
- Musk × Munger: 代码安全审查
- 任意两人/多人: 只要对产品有利就讨论

### CEO 介入条件
- 方向性分歧（做不做某个功能）
- 资源冲突（两个人需要同一个东西）
- 对外发布决策（什么时候发 Reddit/PH）
- 紧急事件（系统崩溃、用户投诉）
